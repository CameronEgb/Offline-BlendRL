## 1. Project Overview
BlendRL jointly learns symbolic (logic) and neural policies. It integrates logic agents (NSFR/Neumann) with RL (PPO, IQL) to create hybrid agents.

## 2. The Golden Rule of Comparison
The ultimate objective of this codebase is to compare **Online** vs. **Offline** learning efficiency.
- **The Rule:** For any data point (x-value) on a comparison plot, the Offline agent must be trained *only* on the exact dataset generated by the Online agent up to that same x-value.
- **X-Axis:** Represented by the number of steps/transitions in the database.
- **Interval Logic:** `INTERVALS_COUNT` defines snapshot points. 
- **Evaluation Standard:** All interval evaluation points (both online and offline) MUST be calculated as a **100-episode average reward** to ensure statistical significance.

## 3. Execution Pipeline & Configuration
The project uses a modular configuration system.

> **CRITICAL ENTRY POINT:** Always use `./run_full_cycle.sh` (or `bin/run_full_cycle.sh`) as the main entry point, NOT `run_full_cycle.py`. The shell script handles virtual environment activation, PYTHONPATH setup, and config list batching.

### Modular Specification
- **Configuration:** All experiment parameters (hyperparameters, environment, methods) are defined in YAML files located in `in/config/`.
- **Precedence:** CLI arguments > YAML config > Env variables > Defaults. This allows overriding specific values from a base YAML template.
- **Batching:** `bin/run_full_cycle.sh` allows specifying a list of config files to run multiple experiments sequentially or in parallel.

### Dataset & Experiment Mapping
- **Experiment ID:** A unique string identifying the run.
- **Run Name:** Constructed as `{env_name}_{method}_{experiment_id}`.
- **Standard Paths:**
    - Experiments: `results/experiments/[GROUP/][EXP_ID]/[RUN_NAME]`
    - Datasets: `results/datasets/[GROUP/][EXP_ID]/[RUN_NAME]`
- **Offline Linkage:** Offline training scripts find their datasets by looking for the `RUN_NAME` derived from the `offline_datasets` list provided in the config/CLI.

## 5. Operational Challenges & Environment
- **Execution Mode (`--local`):** 
    - **Local (`--local`):** Runs experiments as standard background processes. Used for debugging or running on local machines/Macs.
    - **Cluster (Default):** Submits jobs to the research cluster using Slurm (`sbatch`).
- **Organization (`--group`):** The group flag is used when running multiple experiments at once to keep related results together under a single subfolder in `results/`.
- **Path Redirection:** `use_large_dataset_path` redirects `results/datasets/` to high-capacity storage.
- **Syncing:** `sync_logs.sh` pulls `out/experiments/` from cluster -> local.
- **State Recovery:** For long Atari runs (e.g., Seaquest), use the `--recover` flag to resume training from the latest checkpoint in `results/experiments/[EXP_ID]/[RUN_NAME]/checkpoints/`. Training states are saved at every evaluation interval.

## 6. Mandates & Debugging Heuristics
- **Debugging Failures:** 
    - **Primary Resource:** `results/logs/[EXP_ID]/` contains the `.err` and `.out` files. Always check these first for crashes.
    - **Training Audit:** `results/experiments/[EXP_ID]/hyperparameters.txt` provides a summary of the configuration used for the run.
- **Rule Changes vs. Code Changes:** 
    1. Check if the rules work in other environments. If they do, focus on the environment-specific valuation code.
    2. **MANDATE:** **Always ASK** before modifying logic rules (`in/rules/[env]/[ruleset]/`). Prioritize fixing Python code unless the rules are fundamentally flawed for that specific task.
- **Surgical Updates:** Do not refactor unrelated code.
- **Verification:** Ensure `run_full_cycle.sh` orchestration is never broken.

## 7. Symbolic Logic & Environment Definitions
Hybrid agents (BlendRL) require both Python environment wrappers and symbolic logic rules.

### Symbolic Logic Rules
Located in `in/rules/[ENVIRONMENT]/[RULESET]/`:
- `preds.txt`: Definitions of logical predicates.
- `neural_preds.txt`: Mappings of neural network outputs (e.g., object detectors) to predicates.
- `clauses.txt`: The symbolic rules (clauses) used for reasoning.
- `consts.txt`: Constants used in the rules.

### Environment Wrappers
Located in `in/envs/[ENVIRONMENT]/`:
- `env_vectorized.py`: Optimized vectorized environment wrappers for training.
- `blenderl_reward.py`: Custom reward shaping for hybrid agents (if applicable).
